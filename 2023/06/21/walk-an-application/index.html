遍历应用程序–tryhackme

任务 1 遍历应用程序

  • 在这个房间里，您将学习如何仅使用浏览器中的内置工具手动检查 Web 应用程序的安全问题。通常情况下，自动化安全工具和脚本会遗漏许多潜在漏洞和有用信息。

    以下是您将在整个会议室中使用的内置浏览器工具的简短分类：

    • 查看源代码 - 使用浏览器查看网站的人类可读源代码。
    • 检查器 - 了解如何检查页面元素并进行更改以查看通常被阻止的内容。
    • 调试器 - 检查和控制页面 JavaScript 的流程
    • 网络 - 查看页面发出的所有网络请求。

任务 2 浏览网站

  • 作为渗透测试人员，您在审查网站或 Web 应用程序时的职责是发现可能存在漏洞的功能并尝试利用它们来评估它们是否存在。这些功能通常是网站的一部分，需要与用户进行一些交互。

    查找网站的交互部分就像发现登录表单以手动查看网站的 JavaScript 一样简单。一个很好的起点就是用您的浏览器浏览网站并记下各个页面/区域/功能，并为每个页面/区域/功能做一个总结。

任务 3 查看页面源

  • 页面源代码是每次我们发出请求时从 Web 服务器返回给我们的浏览器/客户端的人类可读代码。

    返回的代码由 HTML（超文本标记语言）、CSS（层叠样式表）和 JavaScript 组成，它告诉我们的浏览器要显示什么内容，如何显示它并添加与 JavaScript 的交互元素。

    出于我们的目的，查看页面源可以帮助我们发现有关 Web 应用程序的更多信息。

    如何查看页面源？

    1. 查看网站时，您可以右键单击该页面，然后您会在菜单上看到一个显示“查看页面源代码”的选项。
    2. 大多数浏览器都支持将 view-source: 放在 URL 的前面，例如 view-source:https://www.google.com/
    3. 在浏览器菜单中，您会找到查看页面源代码的选项。此选项有时可以在子菜单中，例如开发人员工具或更多工具。

    让我们查看一些页面源代码！

    尝试查看 Acme IT 支持网站主页的页面源代码。不幸的是，解释您在这里看到的一切超出了这个房间的范围，您需要查看网站设计/开发课程才能完全理解它。我们能做的，就是挑选出对我们很重要的信息。

    在页面顶部，您会注意到一些以 <!-- 开头并以 --> 结尾的代码，这些是注释。注释是网站开发人员留下的消息，通常是为了向其他程序员解释代码中的某些内容，甚至是对自己的注释/提醒。这些评论不会显示在实际网页上。此评论描述了主页是如何在开发新主页时是临时的。查看评论中的网页以获得您的第一个标志。

    HTML 中不同页面的链接写入锚标记（这些是以 <a 开头的 HTML 元素），并且您将定向到的链接存储在 href 属性中。

    例如，您将在第 31 行看到联系页面链接：

    如果您进一步查看页面源代码，会发现一个隐藏链接，指向以“secr”开头的页面，查看此链接即可获取另一个标志。显然，在现实情况下您不会获得标志，但您可能会发现企业使用一些私人区域来存储公司/员工/客户信息。

    可以使用 HTML 代码包含 CSS、JavaScript 和图像等外部文件。在此示例中，您会注意到这些文件都存储在同一目录中。如果您在 Web 浏览器中查看此目录，则存在配置错误。应显示的是空白页面或 403 禁止页面，其中包含错误，指出您无权访问该目录。相反，目录列表功能已启用，实际上列出了目录中的每个文件。有时这不是问题，目录中的所有文件都可以安全地被公众查看，但在某些情况下，备份文件、源代码或其他机密信息可能存储在此处。在本例中，我们在 flag.txt 文件中获得一个标志。

    如今，许多网站都不是从头开始制作的，而是使用所谓的框架。框架是预制代码的集合，允许开发人员轻松包含网站所需的常见功能，例如博客、用户管理、表单处理等等，从而节省开发人员数小时或数天的开发时间。

    查看页面源代码通常可以为我们提供线索，了解某个框架是否正在使用，如果使用，是哪个框架，甚至是什么版本。了解框架和版本可能是一个强有力的发现，因为框架中可能存在公共漏洞，并且网站可能没有使用最新版本。在页面底部，您将找到有关正在使用的框架和版本的评论以及该框架网站的链接。查看该框架的网站，您会发现我们的网站实际上已经过时了。阅读更新通知并使用您找到的信息来发现另一个标志。

任务 4 开发人员工具 - 检查器

  • 开发者工具

每个现代浏览器都包含开发工具；这是一个用于帮助 Web 开发人员调试 Web 应用程序的工具包，让您可以窥视网站的内部结构以了解正在发生的事情。作为渗透测试人员，我们可以利用这些工具来更好地理解 Web 应用程序。我们特别关注开发人员工具包的三个功能，Inspector、Debugger 和 Network。

打开开发者工具

每个浏览器访问开发人员工具的方式都不同。如果您不确定如何访问它，请单击此任务右上角的“查看站点”按钮以获取有关如何访问适用于您的浏览器的工具的说明。

检验员

页面源并不总是代表网页上显示的内容；这是因为 CSS、JavaScript 和用户交互可以更改页面的内容和样式，这意味着我们需要一种方法来查看此时浏览器窗口中显示的内容。 Element inspector 通过向我们提供网站上当前内容的实时表示来协助我们。

除了查看这个实时视图，我们还可以编辑页面元素并与之交互，这有助于 Web 开发人员调试问题。

在 Acme IT 支持网站上，单击进入新闻部分，您将在其中看到三篇新闻文章。

前两篇文章是可读的，但第三篇文章已被屏蔽，内容上方有一条浮动通知，说明您必须成为高级客户才能查看该文章。这些阻挡页面内容的浮动框通常被称为付费墙，因为它们在您希望看到的内容前面竖起了一堵隐喻的墙，直到您付费为止。

右键单击高级通知（付费专区），您应该能够从菜单中选择“检查”选项，该选项将根据您的浏览器或偏好在底部或右侧打开开发人员工具。您现在将看到构成网站的元素/HTML（类似于下面的屏幕截图）。

（点击图片放大）

找到类为 premium-customer-blocker 的 DIV 元素并单击它。您将在样式框中看到所有适用于此元素的 CSS 样式，例如 margin-top: 60px 和 text-align: center 。我们感兴趣的样式是 display: block 。如果您单击单词 block ，您可以键入您自己选择的值。尝试键入 none ，这将使框消失，显示其下方的内容和一个标志。如果该元素没有显示字段，您可以单击最后一个样式下方并添加您自己的样式。试一试元素检查器，您会发现可以更改网站上的任何信息，包括内容。请记住，这只是在您的浏览器窗口中编辑的，当您按下刷新时，一切都会恢复正常。

任务 5 开发人员工具 - 调试器

  • 开发者工具 - 调试器

    开发人员工具中的这个面板旨在用于调试 JavaScript，对于想要找出某些内容可能无法正常工作的 Web 开发人员来说，这又是一项出色的功能。但作为渗透测试人员，它为我们提供了深入挖掘 JavaScript 代码的选项。在 Firefox 和 Safari 中，此功能称为 Debugger，但在 Google Chrome 中，它称为 Sources。

    在 Acme IT 支持网站上，单击联系人页面，每次加载页面时，您可能会注意到屏幕上快速闪烁红色。我们将使用调试器来确定这个红色闪光是什么，以及它是否包含任何有趣的东西。作为渗透测试人员，调试红点不会是您在现实世界中要做的事情，但它确实允许我们使用此功能并习惯调试器。

    在这两种浏览器中，在左侧，您会看到当前网页正在使用的所有资源的列表。如果单击资产文件夹，您将看到一个名为 flash.min.js 的文件。单击此文件会显示 JavaScript 文件的内容。

    很多时候，当查看 javascript 文件时，您会注意到所有内容都在一行上，这是因为它已被最小化，这意味着所有格式（制表符、空格和换行符）都已被删除以使文件更小。该文件也不例外，而且它也被混淆了，故意使其难以阅读，因此其他开发人员无法轻松复制它。

    我们可以使用“Pretty Print”选项返回一些格式，该选项看起来像两个大括号 { } 以使其更具可读性，尽管由于混淆，仍然很难理解文件中发生的情况。如果滚动到 flash.min.js 文件的底部，您将看到以下行：flash['remove']();

    （点击图片放大）

    这一点 JavaScript 就是从页面中删除红色弹出窗口的原因。我们可以利用调试器的另一个功能，称为 断点.这些是代码中我们可以强制浏览器停止处理 JavaScript 并暂停当前执行的点。

    如果单击包含上述代码的行号，您会发现它变成蓝色；您现在已在此行上插入了一个断点。现在尝试刷新页面，您会发现红色框保留在页面上而不是消失，并且它包含一个标志。

任务 6 开发人员工具 - 网络

  • **开发人员工具 - 网络
    **

    开发人员工具上的网络选项卡可用于跟踪网页发出的每个外部请求。如果单击“网络”选项卡然后刷新页面，您将看到该页面请求的所有文件。

    尝试在联系页面上执行此操作；如果列表过于拥挤，您可以按垃圾桶图标删除列表。

    在网络选项卡打开的情况下，尝试填写联系表格并按 发信息按钮。您会注意到网络选项卡中的一个事件，这是使用 AJAX 方法在后台提交的表单。 AJAX是一种在Web应用程序后台通过改变当前网页来不干扰地发送和接收网络数据的方法。

    检查联系表单创建的网络选项卡上的新条目，并查看数据发送到的页面以显示标志。
